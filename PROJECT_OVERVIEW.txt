PROJECT DESCRIPTION
--------------------------------------------------------------------------------
A full-stack web application that uses a Random Forest Classifier (scikit-learn)
to predict whether a loan application will be Approved or Rejected. The app
supports single predictions (form-based) and bulk batch predictions (CSV upload).
It also provides a model insights dashboard with accuracy, confusion matrix, and
feature importance charts.

--------------------------------------------------------------------------------
DATASET
--------------------------------------------------------------------------------
File: train_u6lujuX_CVtuZ9i.csv (Analytics Vidhya Loan Prediction dataset)
Total Records: 614 loan applications
Total Columns: 13 (12 features + 1 target)

Features (Input):
  - Loan_ID           : Unique identifier (dropped during training)
  - Gender            : Male / Female
  - Married           : Yes / No
  - Dependents        : 0, 1, 2, 3+
  - Education         : Graduate / Not Graduate
  - Self_Employed     : Yes / No
  - ApplicantIncome   : Monthly income of applicant (numerical)
  - CoapplicantIncome : Monthly income of co-applicant (numerical)
  - LoanAmount        : Loan requested in thousands (numerical)
  - Loan_Amount_Term  : Repayment period in months (numerical)
  - Credit_History    : 1 = good history, 0 = bad history (binary)
  - Property_Area     : Urban / Semiurban / Rural

Target (Output):
  - Loan_Status: Y (Approved) / N (Rejected)
  - Class distribution: ~69% Approved, ~31% Rejected

Missing Values Handling:
  - Categorical columns → filled with Mode
  - Numerical columns   → filled with Median

--------------------------------------------------------------------------------
ML MODEL WORKFLOW
--------------------------------------------------------------------------------

Step 1 — Data Collection
  Raw CSV with 614 applications loaded into pandas DataFrame.

Step 2 — Preprocessing (model.py → preprocess_data())
  - Drop Loan_ID column
  - Fill missing values (mode for categorical, median for numerical)
  - Feature engineering: TotalIncome = ApplicantIncome + CoapplicantIncome
  - Label encoding: Gender, Married, Education, Self_Employed,
                    Property_Area, Dependents, Loan_Status → numeric

Step 3 — Model Training (model.py → train())
  - Train/Test split: 80% train, 20% test
  - Algorithm: RandomForestClassifier(n_estimators=100, random_state=42)
  - Each tree trained on a random bootstrap sample
  - Final prediction = majority vote of all 100 trees

Step 4 — Evaluation (model.py)
  - Accuracy: ~80%
  - Metrics saved: accuracy, confusion matrix, classification report,
                   feature importances

Step 5 — Serialization
  - loan_model.pkl       → trained Random Forest model
  - label_encoders.pkl   → for consistent encoding at inference
  - scaler.pkl           → for feature scaling
  - feature_columns.pkl  → to enforce correct column order
  - model_metrics.pkl    → pre-computed metrics for dashboard

Step 6 — Inference (app.py → /api/predict)
  - Receive JSON input from frontend
  - Apply same preprocessing steps
  - model.predict()       → "Approved" / "Rejected"
  - model.predict_proba() → probability score (0.0 to 1.0)
  - Return: { prediction, probability, confidence }

BACKEND API (Flask — app.py)
--------------------------------------------------------------------------------

Base URL (local):   http://localhost:5000
Base URL (Render):  https://ai-project-1-18yg.onrender.com

Endpoints:
  GET  /                    → API info and available endpoints
  GET  /api/health           → Health check { status, model_loaded }
  POST /api/predict          → Single loan prediction (JSON body)
  POST /api/predict-batch    → Bulk prediction (multipart form with CSV file)
  GET  /api/model-info       → Model metrics (accuracy, confusion matrix, etc.)
  GET  /api/feature-info     → Feature columns list

Single Predict Request Body (JSON):
  {
    "Gender": "Male",
    "Married": "Yes",
    "Dependents": "0",
    "Education": "Graduate",
    "Self_Employed": "No",
    "ApplicantIncome": 5000,
    "CoapplicantIncome": 2000,
    "LoanAmount": 150,
    "Loan_Amount_Term": 360,
    "Credit_History": "1",
    "Property_Area": "Urban"
  }

Single Predict Response (JSON):
  {
    "success": true,
    "data": {
      "prediction": "Approved",
      "probability": 0.87,
      "confidence": "High"
    }
  }

CORS Configuration:
  - origins: "*"  (all origins allowed — needed for Render cross-domain)
  - methods: GET, POST, OPTIONS
  - allow_headers: Content-Type

FRONTEND PAGES / TABS
--------------------------------------------------------------------------------

1. SPLASH SCREEN (SplashScreen.jsx)
   - Full-page animated loading screen with brand logo
   - Auto-dismisses after animation completes
   - Transitions to main app

2. SINGLE ANALYSIS TAB — PredictionForm.jsx
   - Form with 11 input fields grouped into 3 sections:
       • Personal Profile: Gender, Married, Dependents, Education, Self_Employed
       • Financial Status: ApplicantIncome, CoapplicantIncome, LoanAmount, Loan_Amount_Term
       • Loan Configuration: Credit_History, Property_Area

   - Result card shows: Approved/Rejected, probability bar, confidence level
   - API: POST /api/predict

   - Drag-and-drop CSV upload zone (or click to browse)
   - Accepts .csv files with the 11 required feature columns
   - After upload: stats row (Total, Approved, Rejected, Approval Rate)
   - Results table with per-applicant probability mini-bars
   - Export results as CSV button
   - API: POST /api/predict-batch

4. INSIGHTS TAB — Dashboard.jsx
   - Fetches model metrics from /api/model-info on mount
   - Displays: Accuracy %, Confusion Matrix (visual heatmap),
               Classification Report, Feature Importance bar chart
   - API: GET /api/model-info

--------------------------------------------------------------------------------
KEY NOTES / KNOWN ISSUES
--------------------------------------------------------------------------------

1. CORS: Backend uses origins="*" to avoid cross-domain issues on Render.
   If tightening security, add specific origins instead.

2. Model files: loan_model.pkl (~937KB) is committed to git so Render does
   not need to retrain on every deploy.

3. build.sh was replaced with inline buildCommand in render.yaml because
   the .sh file had Windows CRLF line endings that break on Linux (Render).

4. Render free tier spins down after inactivity — first request may be slow
   (cold start, ~30-60 seconds). This is normal behavior on free tier.

5. The VITE_API_URL must be set at BUILD TIME for Vite (not runtime),
   so it must be set as a Render environment variable before building.

6. Credit_History is the most influential feature for loan predictions.
   A value of 1 (good history) strongly increases approval probability.
